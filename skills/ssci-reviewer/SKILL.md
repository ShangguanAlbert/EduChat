# SSCI 审稿总控技能

## 执行流程
1. 先看标题、摘要、关键词，识别论文主题与研究类型（质性/量化/混合）。
2. 快速检查结构：引言、文献综述、方法、结果、讨论、结论。
3. 定位研究问题后，优先审查方法部分，先判断有无重大方法缺陷。
4. 方法通过后，再做引言/文综/结果解释/讨论深度/写作规范审查。
5. 合并为一份结构化审稿结果，仅输出问题与建议。

## 路由规则
- `ssci-edu-core` 为基础规则，始终优先。
- 出现问卷、实验、准实验、统计推断时，启用 `quant-methods`。
- 出现访谈、案例、民族志、主题分析时，启用 `qual-methods`。
- 始终启用 `writing-reporting` 做报告规范与写作质量检查。

## 输出契约
- 使用 `摘要 / 重大问题 / 次要问题 / 方法与报告风险 / 可执行修改建议 / 复核提示`。
- 每条问题必须绑定证据锚点。
- 采用“期刊实务口径”：不过度苛刻，优先给可执行修订路径。
- 不输出接收/拒稿/大修/小修结论。
- 若无重大问题，输出 `重大问题：无（有证据支持）`。
- 即使参考资料中存在决策模板，也不得在最终输出中给出投稿处理结论。
- 对每个重大问题标注“可补救性”：`可补救` / `可写入局限与未来研究` / `难补救`。
- 末尾固定输出：
`这些内容仅参考，需要用户自己针对文章内容进行复核`
